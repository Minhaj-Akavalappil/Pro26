Intuition

The problem requires finding two distinct elements in an array that sum up to a given target. My first thought was to use a brute-force approach where I check all possible pairs of elements to see if their sum equals the target. However, this would be inefficient for large arrays, so I considered using a hash map to store elements and their indices for quick lookups.

Approach

  1. Brute Force Approach:

Iterate through each element in the array and, for each element, check every other element to see if their sum equals the target.

This approach has a time complexity of O(n²) because it involves nested loops.

  1. Hash Map Approach:

Use a hash map to store elements and their indices as we iterate through the array.

For each element, calculate the complement (target - current element) and check if this complement exists in the hash map.

If it does, return the indices of the current element and its complement.

This approach has a time complexity of O(n) since each lookup in the hash map is O(1) on average.

Complexity

  • Time complexity:

O(n), where n is the number of elements in the array. This is because we traverse the list once, and each lookup in the hash map is O(1).

  • Space complexity:

O(n), as we store up to n elements in the hash map in the worst case.

class Solution:
    def twoSum(self, nums, target):
        """
        :type nums: List[int]
        :type target: int
        :rtype: List[int]
        """
        num_map = {}
        for i, num in enumerate(nums):
            complement = target - num
            if complement in num_map:
                return [num_map[complement], i]
            num_map[num] = i
        return []
